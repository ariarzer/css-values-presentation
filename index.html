Форма Бэкуса-Наура

"text" или 'text' нетерминал
<A> терминал
<A> ::= <B> <A> эквивалентно <B>
<A> <B> конкатенация
<A> | <B> один из элементов
integer имя классса нетерминалов, описывается не через BNF 
            <expression> ::= <add_expr>
            <add_expr> ::= <mul_expr> '+' <add_expr>
                | <mul_expr> '-' <add_expr>
                | <mul_expr>
            <mul_expr> ::= integer '*' <mul_expr>
                | integer '/' <mul_expr>
                | integer '%' <mul_expr>
                | integer
            integer ::= [0-9]+
        
            <add_expr> ::= <mul_expr> '+' <add_expr>
                | <mul_expr> '-' <add_expr>
                | <mul_expr>
=> 
            <add_expr> ::= <mul_expr> [ ('+' | '-') <add_expr> ]
            
        
            <mul_expr> ::= integer '*' <mul_expr>
                | integer '/' <mul_expr>
                | integer '%' <mul_expr>

=> 
            <mul_expr> ::= integer [ ('*' | '/' | '%') <mul_expr> ]

            <mul_expr> ::= integer [ ('*' | '/' | '%') <mul_expr> ]

Value Definition Syntax

  1. ключевые слова, которые записываются буквально, без кавычек
    2.                  initial
  2. базовые типы данных, которые отображаются между '<' и '>'
    3.                   <length>
  3. типы, которые именуются по свойству
    4.                   <'background-attachment'>
[<A> | <B>] <C> группировка
<A> || <B> || <C> один или более из группы в заданном порядке
<A> && <B> && <C> один или более из группы в любом порядке
<A>* ноль или более раз
<A>+ один или более раз
<A>? необязательный
<A> {n} А повторяется n раз
<A> {n, m} А повторяется не меньше n раз и не больше m раз
<A># один или более раз, разделенные запятыми 
            <'text-shadow'> = 
              inset? && [ <length>{2,4} && <color>? ] ]# | none
             
            <'border-width'> = 
              [ <length> | thick | medium | thin ]{1,4}
             
            <'font-family'> = 
              [ <family-name> | <generic-family> ]#

Текстовые типы данных

  1. Pre-defined Keywords
  2. Author-defined Identifiers
  3. Quoted Strings: the < string > type
  4. Resource Locators: the < url > type

Pre-defined Keywords

Пишутся буквально, без кавычек.
Интерпретируются в ASCII без учета регистра 

            .class { 
                heigth: initial;
            }
        
            .class { 
                heigth: INITIAL;
            }

Pre-defined Keywords

  1. CSS-wide keywords - набор значений, общих для всех css-свойств.
            .class { 
                heigth: initial;
                widht: unset;
                margin: inherit;
            }

Pre-defined Keywords

  1. Property-specific keyword - набор значений, определенных для конкретного свойства.
            .class { 
                text-align: center; 
                border-style: solid; 
            }

В спецификации это выглядит так:

            text-align: center | justify | left | right | start | end

Author-defined Identifiers

Любой допустимый идентификатор CSS, который не будет неверно истолкован как pre-defined keyword в определении значения этого свойства.

            div {
              counter-reset: h1-counter 10;
            }
            h1 {
              counter-increment: h1-counter;
            }
            h1::before {
              content: counter(h1-counter);
            }
        
            h1 {
              animation: colorChange 3s infinite alternate;
            }
            @keyframes colorChange {
              from { color: blue; }
              to { color: green; }
            }

Quoted Strings: the <string> type

Последовательность символов, выделенных двойными или одинарными кавычками.

            div::before {
              content: "ololo";
            }

Resource Locators: the <url> type

            body {
              background-image: url("assets/image.jpeg"); 
            }
            .blurry {
              filter: url("#blurFilter");
            }

Числовые типы данных

  1. Integers: the <integer> type
  2. Real Numbers: the <number> type
  3. Numbers with Units: dimension values

Integers: the <integer> type

            +? -? <digit>+

<digit> - a code point between U+0030 (0) and U+0039 (9) inclusive.
Например:

            .class {
              margin: -3px;
            }

Real Numbers: the <number> type

            +? -? <digit>* . <digit>+

Например:

            .class {
              margin: -.3px;
            }

Numbers with Units: dimension values

Число, за которым сразу следует идентификатор единицы измерения.

            +? -? <digit>* .? <digit>+ <length>

Например:

            .class {
              margin: -3px;
            }

Distance Units: the <length> type

  1. Relative Lengths

    2. em, ex, rem, %, vm, vmin ...

  2. Absolute Lengths

    3. cm, px, pc, mm, Q ...

Functional Notations

            .class {
              background: url("http://www.example.org/image");
              color: rgb(100, 200, 50 );
              content: counter(list-item) ". ";
              width: calc(50% - 2em);
            }

Calc(). Syntax.

            calc(50% - 2em);

(выражение) => (значение свойства CSS)

1. Скобки обязательны

            calc(50% - 2em);

2. Вложенные скобки тоже поддерживаются

            calc(((100% - (200px * 4 + 10px * 6)) / 6) * 2 + 200px);

3. Вокруг + и - должны быть пробелы

            calc(50% - 2em);

Calc(). Computed Value.

Если выражение содержит только числа, но вычисляется при парсинге

            calc(1 + 2.5)
               // => 3.5 после парсинга

Calc(). Computed Value.

Если выражение содержит что-то кроме чисел, то вычисляется при первом обращении к свойству

            calc(100% - 100% + 1em)
               // => calc(1em + 0%) после парсинга 
               // => 1em при обращении

Calc(). Computed Value.

Если выражение содержит что-то кроме чисел, то вычисляется при первом обращении к свойству

            width: calc(5px - 10px);
                // => calc(- 5px) после парсинга 
                // => 0px при обращении

Calc(). Type Checking.

Умножение

Хотя бы один операнд должен быть number 
            calc(10% * 10px)
            calc(10% * 10) // => 100%

Calc(). Type Checking.

Деление

Правый операнд number должен быть number 
            calc(100% / 3px)
            calc(100% / 3) // => 33.333%

Calc(). Type Checking.

Нельзя использовать в одном выражении длину и время.

            calc(0 * 5px + 10s)

Нельзя использовать в одном выражении длину или время и число.

                    calc(5px + 0)
                    calc(5s + 0)

Calc(). Range Checking.

            width: calc(5px - 10px);
                // => calc(- 5px) => 0px

Разные ситуации:

                    width: -5px;
                        // property range check
                
                    width: calc(5px - 10px);
                        // calc range check

Итого

  • В CSS есть типизация
  • Если calc() работает не так, как вы ожидаете, проверьте типы.
  • Синтаксис большинства языков программирования описывает EBNF, но...
  • ...для описания типов данных в CSS есть свой Value Definition Syntax.
  • Его знание облегчает упрощает чтение спецификаций.
